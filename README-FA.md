# زنجیر - نسخه بهینه‌شده برای شبکه داخلی، اینترانت و نصب آفلاین

زنجیر در این نسخه برای سرورهایی آماده شده است که به اینترنت بین‌المللی دسترسی ندارند یا باید کاملا در محیط‌های Air-gapped، اینترانت سازمانی، دیتاسنترهای داخلی و شبکه ملی اجرا شوند. بسته آفلاین پروژه همه چیز لازم برای راه‌اندازی را از قبل جمع می‌کند تا نصب روی سرور مقصد بدون نیاز به Docker Hub، GitHub یا Let's Encrypt انجام شود.

**لینک مخزن اصلی:** [https://github.com/sina99778/Zanjir](https://github.com/sina99778/Zanjir)

## ویژگی‌های کلیدی

- نصب‌گر گرافیکی ترمینالی با `whiptail` که راه‌اندازی را مرحله‌به‌مرحله انجام می‌دهد.
- بارگذاری کامل Docker imageها از پوشه `images/` بدون نیاز به اینترنت خارجی.
- استفاده از Caddy با `tls internal` برای ساخت خودکار گواهی داخلی و حذف وابستگی به Let's Encrypt.
- ابزار مدیریتی `zanjir` با منوی تعاملی و فرمان `zanjir doctor` برای عیب‌یابی خودکار.
- معماری Docker Compose بازطراحی‌شده که در آن Caddy تنها ورودی HTTP/HTTPS است و سرویس‌ها پشت شبکه داخلی Compose قرار می‌گیرند.

## روش نصب جدید

### 1. دریافت بسته آفلاین

به صفحه Releases مخزن بروید و فایل `zanjir-offline.tar.gz` را دانلود کنید.

### 2. انتقال به سرور مقصد

فایل دانلودشده را با هر روش مورد نظر خود به سرور منتقل کنید. برای مثال می‌توانید از فلش، شبکه داخلی، `scp` یا هر روش انتقال فایل سازمانی استفاده کنید.

### 3. استخراج بسته

روی سرور مقصد این دستورات را اجرا کنید:

```bash
tar -xzf zanjir-offline.tar.gz
cd offline-bundle
```

### 4. اجرای نصب‌گر

نصب‌گر جدید را با دسترسی ریشه اجرا کنید:

```bash
sudo bash install.sh
```

پس از اجرا:

- نصب‌گر TUI با `whiptail` باز می‌شود.
- در صورت وجود پوشه `images/`، همه imageها به‌صورت آفلاین با نوار پیشرفت بارگذاری می‌شوند.
- از شما دامنه یا IP، پورت HTTPS و ایمیل مدیر پرسیده می‌شود.
- فایل `.env` ساخته یا به‌روزرسانی می‌شود.
- سرویس‌ها بالا می‌آیند و ابزار `zanjir` به‌صورت سراسری در `/usr/local/bin/zanjir` نصب می‌شود.

## معماری جدید

در نسخه آفلاین:

- `caddy` گواهی را با `tls internal` تولید می‌کند و نیازی به دسترسی به Let's Encrypt ندارد.
- مسیرهای `/_matrix/*` و `/_synapse/client/*` به سرویس `conduit` می‌روند.
- مسیر `/admin/*` به سرویس `admin` می‌رود.
- بقیه درخواست‌ها به `element-web` پروکسی می‌شوند.
- فقط Caddy روی پورت‌های HTTP و HTTPS در دسترس میزبان قرار می‌گیرد.

## راهنمای CLI

پس از نصب، می‌توانید از هر نقطه‌ای در سرور این دستور را اجرا کنید:

```bash
zanjir
```

این دستور یک منوی تعاملی رنگی باز می‌کند که شامل گزینه‌های زیر است:

- نمایش وضعیت سرویس‌ها
- مشاهده لاگ‌ها
- ری‌استارت سرویس‌ها
- اجرای Doctor
- خروج

### اجرای مستقیم فرمان‌ها

```bash
zanjir status
zanjir logs
zanjir logs caddy
zanjir restart
zanjir doctor
```

## عیب‌یابی با `zanjir doctor`

فرمان زیر برای بررسی خودکار وضعیت نصب طراحی شده است:

```bash
zanjir doctor
```

این فرمان موارد زیر را بررسی می‌کند:

- نصب بودن Docker و Docker Compose و در دسترس بودن Docker daemon
- وجود فایل `.env` و داشتن متغیرهای حیاتی مثل `DOMAIN`، `HTTP_PORT` و `HTTPS_PORT`
- وضعیت bind شدن پورت‌های HTTP، HTTPS، `3478/udp` و `5349/udp`
- سلامت کانتینرهای `caddy`، `conduit`، `element-web` و `coturn`
- درست resolve شدن دامنه روی IP همین سرور
- شواهد تولید و نگهداری گواهی داخلی در لاگ‌های Caddy

خروجی doctor به‌صورت رنگی و با برچسب‌های `[OK]`، `[WARN]` و `[FAIL]` نمایش داده می‌شود تا ادمین بتواند در محیط‌های آفلاین خیلی سریع مشکل را پیدا کند.

## انتشارهای خودکار

در مخزن یک GitHub Actions workflow اضافه شده که با push کردن tagهایی مثل `v1.0.0` اجرا می‌شود:

- کد را checkout می‌کند
- `pack.sh` را اجرا می‌کند
- فایل `zanjir-offline.tar.gz` را می‌سازد
- آن را به Release همان tag ضمیمه می‌کند

بنابراین برای هر نسخه جدید، بسته آفلاین آماده دانلود از صفحه Releases خواهد بود.

## جمع‌بندی

این نسخه از زنجیر برای شرایطی ساخته شده که اینترنت بین‌المللی در دسترس نیست یا زیرساخت باید کاملا مستقل، داخلی و قابل حمل باشد. اگر هدف شما راه‌اندازی Matrix در شبکه‌های محدود، سازمانی یا ایزوله است، مسیر پیشنهادی همیشه این است:

1. دانلود `zanjir-offline.tar.gz` از Releases
2. انتقال به سرور
3. اجرای `sudo bash install.sh`
4. استفاده از `zanjir` و `zanjir doctor` برای مدیریت و پشتیبانی
